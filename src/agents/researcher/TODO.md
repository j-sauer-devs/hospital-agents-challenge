# Researcher Agent TODOs

The Researcher Agent needs to be implemented as an "Academic Explorer" that synthesizes internal hospital data with global medical research. Following the architectural pattern shown in the [ADK Academic Research Sample](https://github.com/google/adk-samples/tree/main/python/agents/academic-research), this agent should act as an orchestrator within a Multi-Agent ecosystem. The sample is useful as it demonstrates how to implement an "Agent-as-a-Tool" (A2A) approach, connecting specialized sub-agents for distinct tasks (like WebSearch and NewResearch) to execute a structured "analyze-search-propose" workflow. To replicate this, the agent needs a system prompt defining its synthesis role and must be equipped with tools or sub-agents to access internal patient trends (via existing RAG capabilities) and external contemporary literature (via a new Google Search tool integration).

To effectively utilize the current RAG knowledge base, the Researcher Agent can be configured to use the existing `search_knowledge_base` tool (currently used by the Doctor agent). This allows the Researcher to query the internal hospital records to find local patient trends, instances of specific symptoms, or treatment outcomes. The agent can then use these internal findings as a starting point for its "analyze" phase, passing the clinical observations to its external search capabilities to find corresponding global research, effectively bridging the gap between specific local cases and broader academic literature.
