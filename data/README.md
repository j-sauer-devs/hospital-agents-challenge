# Data Directory

This directory contains all the data used for the RAG agent, including raw source documents, generated datasets, and results.

---

## Subdirectories

-   `/raw`: Contains the original, unstructured medical records (PDFs) that are ingested into Vertex AI Search.
-   `/processed`: Contains outputs from various scripts, such as results.

---

## Key Files

### `/raw`

-   **`medical_record_*.pdf`**
    -   **Purpose**: These are the synthetic patient medical records in PDF format. They serve as the primary source documents for the RAG agent.
    -   **How it's made**: These files are generated by the `scripts/generate_data.py` script.

### `/processed`

-   **`golden_dataset.jsonl`**
    -   **Purpose**: This file contains the "golden" or "ground truth" question-and-answer pairs. Each entry includes a question, a context (from which the answer should be derived), and the expected answer. This dataset is used to measure the agent's performance.
    -   **How it's made**: This file is generated by the `scripts/generate_golden_dataset.py` script, which uses Gemini to create Q&A pairs from the raw medical records.

-   **`eval_results.json`**
    -   **Purpose**: This file contains the detailed results from running the agent . It includes the agent's generated response for each question in the golden dataset, along with the scores for metrics like "groundedness" and "instruction_following".
    -   **How it's made**: This file is generated by the `scripts/run_evaluation.py` script. The script runs the agent against each question in `golden_dataset.jsonl` and saves the agent's performance metrics to this file.
